# Стартовый план на 7-10 дней

## Цель спринта

Получить рабочий сквозной сценарий:

- создать задачу
- пересчитать расписание
- увидеть prime-task в интерфейсе

## Конкретные задачи по ролям

1. Интеллект ассистента
- Доработать `GreedyScheduler` в `apps/api/app/modules/intelligence`.
- Добавить поддержку блокировок задач (`depends_on`) и тесты edge-cases.
- Подготовить endpoint-данные для Today view.

2. Frontend
- Реализовать страницы `Today` и `Tasks` в `apps/web`.
- Подключить вызовы: `GET /v1/tasks`, `POST /v1/tasks`, `GET /v1/schedule/today`, `POST /v1/schedule/rebuild`.
- Добавить кнопку "Перепланировать".

3. База знаний
- Перевести `InMemoryKnowledgeRepository` на PostgreSQL-репозиторий.
- Добавить SQL-схему и миграцию для `tasks`, `events`, `task_dependencies`.
- Сохранить совместимость API DTO.

4. Интеграции
- Завести полноценный один провайдер (рекомендуется Google Calendar import).
- Реализовать sync endpoint без моков.
- При импорте событий помечать расписание как dirty.

## Контрольные точки

- День 2: все участники поднимают проект локально.
- День 4: backend API и frontend связаны на тестовых данных.
- День 7-10: demo полного сценария и merge в `develop`.

## Definition of Done для задачи

- Код в своей feature-ветке.
- Обновлены тесты для измененного модуля.
- PR содержит шаги проверки.
- Нет сломанных контрактов API.
